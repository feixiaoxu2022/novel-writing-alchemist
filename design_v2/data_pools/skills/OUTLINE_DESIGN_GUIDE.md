# 大纲设计指南

> 本文档指导如何设计outline.json的三层结构。大纲是故事的骨架——骨架粗糙，正文必然空洞。

## 为什么需要三层结构

看一个典型的"粗糙大纲"：

```json
{
  "act_one": {
    "description": "建立世界观，主角登场，两人相遇",
    "key_chapters": [
      {
        "chapter_number": 1,
        "core_event": "主角在副本中觉醒能力",
        "key_scenes": ["觉醒的瞬间", "第一次使用能力", "遇到女主"]
      }
    ]
  }
}
```

这种大纲的问题：**创作时等于从零开始**。"觉醒的瞬间"到底是什么？在哪里？谁在场？发生了什么具体的事？创作者拿到这个大纲，必须重新构思所有细节，大纲形同虚设。

三层结构就是为了解决这个问题：

| 层级 | 回答什么问题 | 粒度 |
|---|---|---|
| story_synopsis | 这一幕**讲了什么故事**？ | 500-1000字，完整的故事线 |
| chapter_synopsis | 这一章**怎么推进故事**？ | 200-400字，起承转合 |
| scene_sequence | 这一章**具体发生了什么**？ | 每个场景有地点、人物、动作、冲突、结果 |

## 1. 第一层：story_synopsis（幕级故事梗概）

### 1.1 什么是好的story_synopsis

story_synopsis是**用自然语言讲这一幕的故事**，不是一句话标题，不是事件列表，而是一段完整的叙述。

**错误示范**（一句话概括）：
```
"第一幕：建立世界观，男女主相遇，初步产生好感"
→ 信息量为零，"建立世界观"不是故事，"产生好感"不是具体事件
```

**错误示范**（事件列表）：
```
"1.男主觉醒 2.进入副本 3.遇到女主 4.一起通关 5.产生好感"
→ 只有事件骨架没有故事逻辑，看不出为什么A导致了B
```

**正确示范**（故事梗概）：
```
刑警陆沉在一次例行调查中意外觉醒了"死亡倒计时"能力——他能看到身边每个人头顶
显示的死亡时间。起初他以为这是幻觉，直到他看到未婚妻苏清禾头顶的文字不是倒计时，
而是一行字："将在今晚23:00杀害你"。

恐惧让陆沉开始暗中调查苏清禾的行踪。他发现她最近频繁接触一个神秘组织，手机里有
加密通讯记录。他试图通过转移关键证人来改变命运，却发现自己的每一个举动都在系统的
预测之内——转移证人反而让苏清禾提前启动了杀人计划。

第一幕在陆沉试图软禁苏清禾时达到转折：他在她的包里发现了准备好的凶器。他意识到
自己的防备不但没有阻止死亡，反而加速了它的到来。预测之所以不可更改，是因为他的
每一次反抗本身就是预测的一部分。
```

### 1.2 写作要点

story_synopsis必须包含：

1. **主角的起始状态**：这一幕开始时主角在做什么、处于什么处境
2. **核心事件链**：发生了什么事，因果关系是什么（因为A所以B，B导致了C）
3. **主角的行动和决策**：主角做了什么选择，为什么这样选
4. **状态变化**：这一幕结束时主角的处境/心态与开头有什么不同
5. **转折铺垫**：为什么故事在这里转向了下一幕

## 2. 第二层：chapter_synopsis（章节故事梗概）

### 2.1 什么是好的chapter_synopsis

chapter_synopsis是**一个章节的故事逻辑**，描述"这一章从什么状态开始，经过什么冲突，到达什么状态"。

**错误示范**（重复core_event）：
```
chapter_synopsis: "陆沉发现苏清禾的秘密"
→ 这是一句事件描述，不是故事逻辑
```

**正确示范**：
```
chapter_synopsis: "陆沉带着怀疑回到家中（起始：表面平静内心恐惧），苏清禾一如
既往地准备晚餐。陆沉借口找东西翻看了苏清禾的包，发现了手术刀和药剂（冲突爆发：
证据确凿），他的手开始发抖但强迫自己镇定。苏清禾从厨房出来时注意到他的异样，
追问怎么了（张力升级：被发现的危险）。陆沉撒谎说只是工作压力，但他知道苏清禾
已经起了疑心。当晚两人各怀心思地躺在同一张床上（结束状态：从单方面怀疑变成
双方互相试探的危险平衡）。"
```

### 2.2 写作要点

chapter_synopsis必须回答三个问题：

1. **起始状态**：本章开始时，主角（和关键角色）处于什么物理/心理状态
2. **推进动力**：本章中发生了什么具体事件推动故事前进
3. **结束状态**：本章结束时，状态发生了什么变化（比开头好了还是糟了？知道了什么新信息？关系发生了什么改变？）

## 3. 第三层：scene_sequence（场景序列）

### 3.1 场景的8个必需字段

每个场景必须具体到可以直接作为创作指导：

| 字段 | 要求 | 常见错误 |
|---|---|---|
| scene_number | 章内序号 | — |
| scene_location | 具体地点环境 | ❌"城市里" ✅"凌晨3点的警局档案室，只有应急灯亮着" |
| characters_present | 在场角色 | 不要写没有戏份的角色 |
| scene_purpose | 在chapter_synopsis中的作用 | ❌"推进剧情" ✅"揭示苏清禾的真实身份，为第3章的信任危机埋伏笔" |
| core_action | 谁对谁做了什么 | ❌"了解情况" ✅"陆沉翻开苏清禾的包，发现底层夹层里藏着手术刀和三支不明药剂" |
| conflict_point | 冲突爆发点 | ❌"产生矛盾" ✅"苏清禾从厨房出来，看到陆沉手里拿着她的包，两人目光对视的瞬间" |
| emotional_shift | 情感从A到B | ❌"紧张" ✅"从'也许是误会'的自我安慰 → '她真的要杀我'的确认恐惧" |
| scene_outcome | 结果+伏笔 | ❌"事情解决了" ✅"陆沉假装没发现，把包放回原处，但苏清禾已经注意到包的拉链方向变了" |

### 3.2 core_action的写法（最关键的字段）

core_action是场景的灵魂。**必须是可视化的具体行为**，读到这句话，脑中应该能浮现出画面。

**判断标准**：能不能画成分镜？

```
❌ 不能画成分镜的：
"两人讨论了计划" → 讨论什么？怎么讨论？坐着还是站着？
"角色A了解了情况" → 怎么了解？看文件？听汇报？偷听？
"关系产生了变化" → 什么行为导致变化？

✅ 能画成分镜的：
"陆沉把窃听器贴在苏清禾手机壳内侧，手指因为紧张而按错了两次"
"苏清禾把手术刀塞进枕头下面，动作熟练得像在整理餐具"
"两人在审讯室隔着单面玻璃对视，陆沉看着她的嘴型念出了'对不起'"
```

### 3.3 场景间的因果链

场景不是独立的事件，前一个场景的outcome必须导致下一个场景的发生：

```
场景1 outcome: "陆沉假装没发现，但苏清禾已经注意到包的拉链方向变了"
   ↓ 因果：苏清禾起了疑心，所以...
场景2 core_action: "苏清禾故意在客厅留下手机，测试陆沉是否会偷看"
   ↓ 因果：陆沉果然偷看了，被苏清禾从监控看到了，所以...
场景3 core_action: "苏清禾将备用凶器从家里转移到车的后备箱"
```

如果去掉任何一个场景，因果链就断了——这说明每个场景都是必要的。

### 3.4 每章至少3个场景的意义

3个场景不是为了凑数，而是确保每章有最基本的**起承转**：

- 场景1（起）：建立本章的初始状态和问题
- 场景2（承）：主角尝试解决问题，遭遇新的复杂性
- 场景3（转）：情况发生关键变化，推向下一章

如果一个章节只需要1-2个场景就能讲完，说明这个章节的信息密度不够，应该合并到相邻章节。

## 4. 转折点设计

### 4.1 什么是好的turning_point

turning_point是两幕之间的关键转折，它必须**不可逆地改变**主角的处境或认知。

**错误示范**（可逆的/没变化的）：
```
turning_point: "主角遇到了困难"
→ 什么困难？遇到困难然后呢？这不是转折，这是日常
```

**正确示范**（不可逆的）：
```
turning_point: "陆沉在苏清禾的包里发现凶器，确认了她要杀自己这个事实。
从此他不再能用'也许是误会'来安慰自己——真相已经无法假装看不见。"
→ 认知被永久改变了，故事不可能回到之前的状态
```

### 4.2 转折点与人物关系的呼应

好的转折点同时改变外部剧情和人物关系：

```
外部转折：发现搭档是卧底
关系转折：从并肩作战 → 不知是敌是友
→ 同一个事件同时推动了剧情和关系，高效且自然
```

## 5. 不同篇幅的大纲策略

| 篇幅 | 章节数 | story_synopsis | chapter_synopsis | scene_sequence |
|---|---|---|---|---|
| 超短篇(6500-10000字) | 3-5章 | 每幕300-500字即可 | 每章150-250字 | 每章2-3个场景 |
| 短篇(15000-40000字) | 6-12章 | 每幕500-800字 | 每章200-350字 | 每章3-4个场景 |
| 中篇(80000-250000字) | 20+章 | 每幕800-1000字 | 每章300-400字 | 每章3-5个场景 |

**注意**：篇幅越长，大纲越重要。中篇没有扎实的大纲，写到中段必然迷失方向。

## 6. 自查清单

| 检查项 | 通过标准 |
|---|---|
| story_synopsis是完整故事吗？ | 能独立讲通，不是标题或事件列表 |
| chapter_synopsis有起始-推进-结束吗？ | 三个问题都能回答 |
| core_action能画成分镜吗？ | 脑中能浮现画面 |
| 场景间有因果链吗？ | 去掉任一场景，后续会断裂 |
| 转折点不可逆吗？ | 主角不可能回到转折前的状态 |
| 每个配角在大纲中有戏份吗？ | characters.json中的角色都出现了 |
| 关系转折与剧情转折呼应吗？ | 同一事件同时改变剧情和关系 |
